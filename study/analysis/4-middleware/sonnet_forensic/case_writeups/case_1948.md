### 3. chaos 机制

ContainerKill 把 ts-preserve-service container 杀 4 秒，K8s 自动重启。物理硬指纹三件套：
1. **k8s.container.restarts**: `ts-preserve-service-5d979f4b55-n6ccw` 0→1（唯一新增；ts-ticket-office chronic 3 不变）
2. **container.cpu.usage**: avg 0.058→0.488（**8.45×**）；JVM cold-start 重新加载 class
3. **hubble_http_p50/p90/p95/p99 全部 NaN**（normal 期 0.0065/0.017/0.028/0.044），service-mesh 无法 observe 任何完成的 HTTP 请求

trace 端 survivorship bias：4 秒短窗口内死掉的请求未被 export，重启后端点重新 200。`ts-preserve-service` 自身 trace status 全 200；503 只出现在 caller `ts-ui-dashboard` 上（17 条 503 of 118 calls）。

### 9. 失败池使用规则

```
本 case 命中标签：A2, A3, B3
最有效组合：A2 + B3（强证据 sticky + 全局 restart counter 差分扫描）
```

A2 处理方式：发现 GT 直接指纹（restart 0→1 / CPU >5× ratio / hubble NaN）后，必须强制 commit 路径，禁止后续单点 outlier 信号反向覆盖；可设计：当 think_tool 出现"the most critical finding yet"+具体 service name 后，强制把该 service 进入"ranked top-1 candidate"，后续假说必须显式比较 ratio 才能取代。

A3 处理方式：max 单点 outlier 锚定前，强制看 distribution（avg / p50 / p95 / p99）；max ratio > 5× 但 avg ratio < 2× → 不是 RC 候选，是 cascade victim 单点 outlier。

B3 处理方式：怀疑某 service 时强制按 metric list SELECT value；额外强化"全局 restart counter 差分扫描"作为每 case 标准步骤，无论调查走到哪个 round 都应在阶段性回顾时跑一次。

### 10. 判断

- **数据是否完整**：✅ restart 0→1、CPU 8.45×、hubble NaN 全部可查；sonnet R36 已发现 CPU 8.45×
- **chaos 是否生效**：✅ ContainerKill 物理 kill，restart counter 直接体现
- **真假盲区分类**：**(b) 框架/模型盲区** — 数据完全可达，sonnet 在 R36 已识别 GT 直接指纹但没 commit；最终被 callees 单点 outlier 假说覆盖
- **失败模式归纳**：A2（R36 GT 直接指纹未 sticky，被 callees outlier 假说覆盖）+ A3（max 单点 outlier 驱动锚定 ts-basic / ts-seat）+ B3（全局 restart counter 差分扫描缺失）
- **共性 vs sonnet-specific**：**部分共性 + 各自不同失败侧**——
  - 共性：两 model 都看到了 GT 直接指纹（sonnet R36 CPU 8.4× / qwen3.5 v4 R46 restart 0→1）但都未 commit；两 model 都被 trace status code 视角主导
  - 不同：sonnet 错向**下游 callees**（cascade 方向倒置），qwen3.5 错向**上游 caller**（ts-ui-dashboard 503 generator）
- **给 sonnet 中间件设计的提示**：A2 + B3 组合——(1) "the most critical finding yet" + service name 出现后强制把该 service 进入 sticky top-1 candidate；(2) 全局 restart counter 差分扫描列为 standard step；(3) 加 cascade 方向校核：root_causes 必须是调用链 leaf 终端被注入处，不可同时出现 callees + 不出现真正 chaos target
