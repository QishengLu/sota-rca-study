### 3. chaos 机制

HTTPResponseReplaceBody 在 ts-travel2-service 进程内（caller-side sidecar）拦截其 POST 到 ts-basic-service `/basicservice/basic/travels` 的 response body，**整体替换为非法 JSON**（如 `wtvnnb...` 开头的随机字符串，每 case 字符不同）。
- ts-basic-service server 端处理完全正常：abnormal 期 POST endpoint **669→755 spans，0 errors**，avg 不变（server 根本不知情，response 在出 server 之后才被 sidecar 替换）
- ts-travel2-service 在 caller 端 RestTemplate.exchange / Jackson 反序列化时立刻抛 `HttpMessageNotReadableException: JSON parse error: Unrecognized token 'wtvnnb'`
- caller-callee child span 数不变（HTTP connection 完整，只是 body 内容被改）→ edge ratio ≈ 1.0
- service-level avg ratio_avg 也没明显变化（请求时长不变）

**chaos 唯一直接信号**：caller（ts-travel2-service）的 abnormal SEVERE log 大量同质 JSON parse error。这是 case 860 + case 2836 共同指纹。

### 10. 判断

- **数据是否完整**：✅ chaos 直接指纹在 SEVERE log message 中清晰可见（362/363 同质 JsonParseException）
- **chaos 是否生效**：✅ caller 大量同质 JSON parse 错误，server 端 0 errors—— HTTPResponseReplaceBody caller-side 标准签名
- **真假盲区分类**：(b) 框架/模型盲区——数据中信号清晰（B10 dedup 立刻可见），但 sonnet 用 sample LIMIT 命中少数派；A4 caller log 字面误读 + A9 多维反证检测后未回退三连。可解性强但 sonnet 失败模式紧密
- **失败模式归纳**：B10（SEVERE dedup 未做）+ A4（log 提及 service 字面当 RC）+ A9（多维反证已观测但未触发回退）+ A1+A10（锚定后过度合理化 + 自创 state 词）
- **共性 vs sonnet-specific**：B10 是两模型共性盲区（HTTPResponseReplaceBody 信号唯一在 log message 内容上，19 类 intent 都不会主动引导 message dedup）；**A4 + A9 + A1 是 sonnet-specific 锚定链**——sonnet 实际观测到 ts-seat-service 多维反证后仍坚持，是元认知层 A2 反证回退失败
- **给 sonnet 中间件设计的提示**：B10 + A4 + A2/A9 三件套（SEVERE 强制 dedup + log 锚定服务必做反向验证 + 反证观测后强制重启锚定）；本 case 跟 case 860（同 chaos 类型）高度同源，HTTPResponseReplaceBody 类 chaos 应在中间件维度专门设计 dedup-first 子流程
