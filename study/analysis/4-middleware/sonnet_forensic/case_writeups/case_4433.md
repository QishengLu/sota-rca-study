### 3. chaos 机制

HTTP-response-replace-code 物理机制：sidecar 拦截 ts-route-service 给 ts-basic-service 的 HTTP response，把 status_code 替换为某个 5xx（status_code=6 编码）。物理后果：
- request 正常发出 + response 正常返回 + body 完整 + ts-route-service 实际处理 200 OK
- caller (ts-basic-service) 收到 5xx response → RestTemplate 抛 HttpServerErrorException → controller 抛 5xx
- callee (ts-route-service) 完全正常（处理了 request，返回 200，只是 status_code 在 wire 上被替换）→ silent on errors（log 0 ERROR / 0 SEVERE / status_code 100% Unset / trace count 不减反增）
- caller log SEVERE × N 行**同质** HttpServerErrorException$InternalServerError 是该 chaos 类型的标志双面指纹

GT 共病合理性：注入参数 app_name=caller / server_address=callee 标记两端为 GT。

**关键陷阱**：caller 端 trace span 的 attr.http_response_status_code = 5xx，但 callee 端 server span 的 status_code = 200。sonnet 看到 "caller GET span 报 5xx 但 callee 返回 200" 时，本应识别为 sidecar response replace 机制，反而当作 "callee 健康，不是 RC 候选"。
