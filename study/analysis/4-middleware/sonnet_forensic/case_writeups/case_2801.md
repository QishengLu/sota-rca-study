### 3. chaos 机制

HTTP-response-replace-body 物理机制：sidecar 拦截 ts-basic-service 给 ts-travel-service 的 HTTP response，把 response body 替换成无效 JSON（body_type=1）。物理后果：
- request 正常发出 + response 正常返回 + status_code 200 → caller-callee child span 数几乎不变（ratio ≈ 1.0）
- caller (ts-travel-service) 的 RestTemplate 收到 response 后做 JSON parse 失败 → 抛 RestClientException "Error while extracting response" → controller 抛 500
- callee (ts-basic-service) 自己处理完了请求，server span 完全正常 → silent on errors（log 0 ERROR / 0 SEVERE / status_code 100% Unset）
- caller log SEVERE × N 行**同质** RestClientException 是该 chaos 类型的标志双面指纹：(a) caller log dedup count 单一 message 大量重复 (b) callee 完全正常

GT 共病合理性：注入参数 app_name=caller / server_address=callee 标记两端为 GT。
