
### 10. 判断

- **数据是否完整**：✅ 双面指纹清晰，service-level 已直接指向 ts-travel-plan-service
- **chaos 是否生效**：✅ delay_duration=4221ms 几乎完美在 caller GET span p50=4229ms 中体现（误差 < 0.2%）
- **真假盲区分类**：(b) 框架/模型盲区——sonnet 在已经看到 ts-travel-plan-service 23x 强信号后主动 pivot 到 worker-node 分析，被 host-level metric 误读链 + max-only outlier 锁死
- **失败模式归纳**：A6（host-level metric 当 pod 信号）触发 A12（node-level 假说未做 cpu_limit 反证）支撑 A3（max-only outlier 锚 ts-seat-service）；B2 + B7 缺失但本 case 不致命（service-level 已足够强）
- **共性 vs sonnet-specific**：B2 + B7 是两模型共性盲区（但 qwen3.5 不依赖即答对）；**A6 + A12 + A3 三件套是 sonnet-specific 失败链**——qwen3.5 没追 worker-node 歧路
- **给 sonnet 中间件设计的提示**：A6 + A12 + A3 联动（host-level metric 类型识别 + node-level 假说 cpu_limit 饱和 gate + max-only 反证）；这条链对 sonnet 的"沿物理拓扑深挖→单点 outlier 锚定"决策模式特别有效
