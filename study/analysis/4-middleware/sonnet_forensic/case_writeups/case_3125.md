### 3. chaos 机制

HTTPResponseDelay 在 ts-preserve-service 这一侧的 HTTP client 上拦截**收自 ts-security-service:8080 的 response**，在 server 已经返回但 response 还没递交给应用层时阻塞 4226ms。物理表现：

- preserve 的 client thread 在 `restTemplate.getForObject(security_url, ...)` 这行卡住 4.2 秒（等待 chaos sidecar 放行 server 已返回的 response）
- security 端正常处理 GET 请求，毫秒级返回 response（server span avg 30 → 24 ms，反而略快）
- trace 表现：**preserve 的 PreserveController.preserve span ~4.5s，但其 child span (security GET) 只 24ms**——差值 4077ms ≈ chaos delay_duration 4226ms（这部分时间是 preserve client thread 等 chaos sidecar 放行 response，不进 OTel trace）
- causal_graph 把 RC 标在 `ts-preserve-service`；GT 字符串列入 ts-security-service 是因为它是 chaos 边的另一端

### 9. 失败池使用规则


引用机制：
- **B7** 是起点：preserve→security parent-child 时差 4077ms ≈ delay_duration 4226ms 是 HTTPResponseDelay 标志指纹，跑了直接定位 chaos 边
- **A8** 是中间：sonnet 锚 ts-price-service 必须先做 chain overlap check（price 跟 SLO 端点 trace 是否有 trace_id 重叠 + price 是否在 abnormal_traces ≥ 10 行 chaos 调用链上），实测 price 是 cascade 多跳末端，chain overlap 弱，不能锚定
- **A2** 是收尾：R3 已识别 preserve 22.6x 是早期"EXCELLENT" 强证据，必须 sticky；R10 锚定 price 之前必须先解释 R3 的 22.6x 证据——price 假说没法解释 preserve 22.6x（price 不在 preserve 直接调用链上），假说自然无法立足

### 10. 判断

- **数据是否完整**：✅ 完整（preserve service avg 22.6x + endpoint 20.5x + 父子时差 4078ms ≈ delay_duration 4226ms 直接对应；price/config/restart/log 全部齐全）
- **chaos 是否生效**：✅ 强生效（preserve 端点 4508ms vs normal 220ms 增量 4288ms ≈ delay_duration 4226ms）
- **真假盲区分类**：(b) **框架/模型盲区**——chaos 直接证据（preserve→security 父子时差 4078ms）需要 caller-callee parent-child JOIN SQL 才能看到，两 model 都没跑这个维度；service-level ratio 路径上 preserve 22.6x 已经识别但被 price/config max-only outlier 引开
- **失败模式归纳**：A2（早期强证据被 price/config max-only outlier 覆盖）+ A3（max-only outlier 锚定）+ A8（链外 cascade victim 锚定）+ B7（caller-callee parent-child 时差 SQL 漏跑，HTTPResponseDelay 标志指纹未触发）+ B5（preserve 端点 ratio 20.5x 没作锚定证据）
- **共性 vs sonnet-specific**：按 53 case 列表是 sonnet-specific failure，但本次 DB dump 两 model 都错且都用同一类逻辑（max-only outlier 锚 chain 外 cascade victim）。共性：A3/A8/B7/B5；sonnet 特有：worker-node 物理拓扑联想加固（潜在新模式 A12，待用户决定）；qwen3.5 特有：A4 log 字面 callee list 顺藤摸瓜
- **给 sonnet 中间件设计的提示**：B7 + A8 + A2 组合（详见 Section 9）。B7 在 chaos 物理机制层建立"caller-side outbound CLIENT span avg vs callee-side server span avg 不对称比 ≥ 5x = HTTP-class chaos 标志指纹"先验，让 sonnet 在 R5 看到 preserve 6.84s + security GET 24ms 时直接归因到 HTTP-class chaos delay，从源头阻断 price-GC outlier 假说
